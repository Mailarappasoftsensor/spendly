● Here's the project structure explained:

  expense-tracker/
  │
  ├── app.py                  # Main entry point — defines all routes and starts
   the Flask server
  │
  ├── requirements.txt        # Lists all Python packages needed to run the
  project
  ├── .gitignore              # Tells Git which files to ignore (venv,
  .DS_Store, etc.)
  │
  ├── database/               # Database layer
  │   ├── __init__.py         # Makes this folder a Python package
  │   └── db.py               # Will contain SQLite connection & table setup
  (not yet written)
  │
  ├── templates/              # HTML pages (rendered by Flask/Jinja2)
  │   ├── base.html           # Master layout — navbar, footer, shared structure
  │   ├── landing.html        # Home/marketing page (/)
  │   ├── login.html          # Login form (/login)
  │   └── register.html       # Register form (/register)
  │
  ├── static/                 # Files served directly to the browser
  │   ├── css/
  │   │   └── style.css       # All custom styling for the app
  │   └── js/
  │       └── main.js         # Client-side JavaScript
  │
  └── venv/                   # Virtual environment (Python dependencies
  installed here)

  How it fits together:

  1. app.py receives a browser request (e.g. /login)
  2. It calls render_template("login.html")
  3. Flask finds login.html in templates/, which extends base.html for the
  shared layout
  4. base.html loads style.css and main.js from static/
  5. The final HTML is sent back to the browser

  The database/ folder is the only part not yet implemented — students build it
  as part of the exercise.
